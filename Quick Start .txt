Quick Start Guide

Installation (2 minutes)

Install dependencies:

pip install -r requirements.txt 

Verify installation:

python -c "import torch; print(f'PyTorch version: {torch.__version__}')" 

Basic Training (5 minutes to start)

Option 1: Default Settings (Recommended for first run)

python enhanced_glyph_recognition.py 

This will:

Download EMNIST dataset automatically (~500MB)

Train on 47 character classes

Run for 50 epochs

Save results to outputs/ directory

Option 2: Quick Test (1-2 minutes)

python enhanced_glyph_recognition.py --epochs 5 --skip-eval 

Option 3: With Learning Rate Finder

python enhanced_glyph_recognition.py --lr-finder

Using Your Trained Model

Predict on a single image:

python inference.py --model outputs/best_model.pth --image my_image.png 

Predict on multiple images:

python inference.py --model outputs/best_model.pth --batch ./my_images/ 

Get visualization:

python inference.py --model outputs/best_model.pth --image my_image.png --visualize

Expected Performance

EMNIST-Balanced (47 classes) - Default

Training time: 1.5-2.5 hours on GPU, 6-8 hours on CPU

Expected accuracy: 88-91% on test set

Model size: ~3.5 MB

EMNIST-ByClass (62 classes)

Training time: Similar to balanced

Expected accuracy: 85-87% on test set

Model size: ~3.8 MB

Tips for Best Results

Use GPU if available: 10-15x faster than CPU

# Check GPU availability python -c "import torch; print(torch.cuda.is_available())" 

Use Learning Rate Finder: Finds optimal LR automatically

python enhanced_glyph_recognition.py --lr-finder 

Monitor training: Watch the training logs for:

Loss should decrease steadily

Validation accuracy should increase

Early stopping will trigger if overfitting

Check results: Look at:

confusion_matrix.png - See which classes are confused

per_class_accuracy.png - Identify weak classes

misclassification_analysis.txt - Common mistakes

Troubleshooting

Issue: CUDA out of memory

Solution: Reduce batch size

python enhanced_glyph_recognition.py --batch-size 64 

Issue: Training too slow on CPU

Solutions:

Reduce epochs for testing:

python enhanced_glyph_recognition.py --epochs 10 

Use smaller batch size to free up memory:

python enhanced_glyph_recognition.py --batch-size 64 

Issue: Model not improving

Solutions:

Run learning rate finder:

python enhanced_glyph_recognition.py --lr-finder 

Check data augmentation (in code):

Too much rotation can hurt performance

Adjust in get_default_config()

Try different scheduler:

Edit config['scheduler'] in code

Options: 'cosine', 'plateau', or None

Issue: Import errors

Solution: Make sure all dependencies are installed

pip install -r requirements.txt --upgrade
